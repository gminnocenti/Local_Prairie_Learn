Criteria for Choosing Which Node to Expand in DFS

In Depth-First Search (DFS), when multiple nodes are connected to the same origin node, there must be a consistent way to choose which node to expand first. In this case, we will always choose to expand the node with the smallest number.

This rule ensures a predictable path of exploration. By selecting the node with the smallest number first, DFS will explore nodes in a consistent and orderly manner, reducing ambiguity in the traversal process.

This choice of prioritizing the node with the smallest number is important, as it keeps the search process structured, especially when different paths are available from the same node. This approach allows DFS to explore each branch fully while adhering to a clear, ordered pattern.
